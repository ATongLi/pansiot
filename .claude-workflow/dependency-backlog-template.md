## 依赖完成通知流程

当依赖模块完成时:

1. **workflow-coordinator** 检测到依赖完成
2. 查询 dependency-backlog.md 找到所有依赖此模块的任务
3. 创建补齐任务并通知

### 通知模板
```
⚠️ 依赖完成通知
依赖模块: IMP-{NUMBER} ({模块名称})
完成时间: {YYYY-MM-DD HH:MM}

影响范围:
- D-{NUMBER}: IMP-{NUMBER} 需要补齐 {内容}
- D-{NUMBER}: IMP-{NUMBER} 需要补齐 {内容}

建议:
1. 按优先级 P0 → P1 → P2 依次补齐
2. 每个补齐后进行集成测试
3. 更新代码映射表
4. 更新 dependency-backlog.md 标记为已完成
```

## 依赖补齐 Checklist

对于每个待补齐的依赖:

### 1. 准备阶段
- [ ] 确认依赖模块已完成并可用
- [ ] 确认依赖模块已通过测试
- [ ] 备份当前 mock 实现的代码

### 2. 补齐阶段
- [ ] 定位依赖忽略标记的代码位置
- [ ] 移除 mock 实现
- [ ] 替换为真实的功能调用
- [ ] 更新错误处理逻辑

### 3. 验证阶段
- [ ] 运行单元测试
- [ ] 运行集成测试
- [ ] 手动测试功能正常
- [ ] 代码审查

### 4. 文档更新
- [ ] 移除代码中的依赖忽略注释
- [ ] 更新 feature-to-code-map.md
- [ ] 更新 dependency-backlog.md 标记为已完成
- [ ] 更新 rt-matrix.md

### 5. 通知相关方
- [ ] 通知依赖此模块的其他开发者
- [ ] 更新项目状态
- [ ] 更新 active-tasks.md
